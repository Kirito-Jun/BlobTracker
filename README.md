# 🧊 Blob Tracker — Liquid Glass Edition

> 基于帧差法的视频运动目标（Blob）实时追踪与可视化特效系统。纯前端项目，零依赖，开箱即用。

## 📋 项目概述

| 属性 | 说明 |
|------|------|
| **名称** | Blob Tracker — Liquid Glass Edition |
| **类型** | 纯前端 Web 应用（零依赖，无需构建工具） |

## ✨ 主要功能模块

### 🎨 1. WebGL 动态背景（ColorBends）

应用底层是一个全屏 WebGL 着色器动画背景，为整个界面提供沉浸式的视觉氛围。

- **原生 WebGL 实现**：不依赖 Three.js 等第三方库，直接使用 WebGL API 编译着色器、绑定缓冲区、驱动渲染循环
- **多主题调色板**：内置 6 套主题色板——Default（默认）、Aurora（极光）、Sunset（日落）、Neon（霓虹）、Ocean（海洋）、Fire（火焰），用户可一键切换
- **鼠标交互**：指针移动产生视差偏移，影响背景的旋转与扭曲变形
- **动态参数**：频率、噪声、缩放、旋转速度等参数随时间动态变化，形成流动的有机图案
- **透明合成**：背景支持 Alpha 混合，确保上层 UI 的 `backdrop-filter` 玻璃态效果能够正确采样底层画面

### 📹 2. 视频源输入

系统支持两种视频输入方式：

#### 2.1 本地上传
- 通过文件选择器上传本地视频文件（支持 MP4、MOV、WebM 等常见格式）
- 使用 `URL.createObjectURL` 创建本地预览流

#### 2.2 实时摄像头
- 调用 `navigator.mediaDevices.getUserMedia` 获取用户摄像头画面
- 默认请求 1280×720 分辨率，前置摄像头
- 纯视频流，不采集音频

#### 2.3 超规格视频检测与降级
- **三级帧率检测**：
  1. **MP4 Box 二进制解析**：读取文件头部/尾部 moov 原子盒，提取 timescale 与 sample_delta，精确计算帧率（零播放延迟）
  2. **requestVideoFrameCallback（rVFC）中位数法**：强制 1x 播放，采样 90 帧，去极值后取中位数，吸附到标准帧率（23.976/24/25/29.97/30/48/50/59.94/60/90/120/144/240fps）
  3. **实时钟帧计数**：以真实挂钟时间 2.5 秒为窗口，计数 rVFC 回调次数，计算平均帧率
- **白名单校验**：
  - 档位 1：分辨率 ≤ 4K（3840×2160）且帧率 ≤ 30fps
  - 档位 2：分辨率 ≤ 1080P（1920×1080）且帧率 ≤ 60fps
- **超规格弹窗**：若视频超出白名单（如 4K60fps），弹出模态框提示用户选择降级方案：
  - 降级到 4K 30帧（画质优先）
  - 降级到 1080P 60帧（流畅优先）
  - 取消导入
- **自动适配**：降级后自动调整跳帧参数，平衡性能与效果

### 🧠 3. Blob 追踪引擎

核心追踪算法基于经典的帧差法，针对浏览器环境进行了大量优化。

#### 3.1 图像处理管线
1. **灰度转换**：将视频帧从 RGBA 转换为灰度图（`Uint8Array`）
2. **高斯模糊**：使用 5×5 高斯核对灰度图进行快速模糊，抑制噪声（可配置跳帧时跳过）
3. **背景差分**：当前帧与前一帧模糊图进行差分，阈值化生成运动掩码
4. **形态学膨胀**：对运动掩码进行 3 像素半径的膨胀操作，连接断裂区域
5. **BFS 轮廓查找**：8 连通域广度优先搜索，提取连通区域边界框
6. **NMS 抑制**：非极大值抑制，移除重叠度过高的冗余框
7. **时域平滑**：对 blob 位置与尺寸进行 0.4 系数的一阶平滑，减少抖动

#### 3.2 性能优化
- **跳帧处理（Skip Frames）**：用户可配置跳帧级别（0-5），降低差分采样密度，提升流畅度
- **计算与渲染解耦**：
  - 追踪计算循环：独立 `setTimeout` 驱动，固定 30fps（33ms 间隔）
  - 渲染循环：`requestVideoFrameCallback`（rVFC）或 `requestAnimationFrame`（RAF）驱动，匹配视频原生帧率（30/60/120fps）
  - 两路完全分离，渲染永不被计算阻塞
- **插值平滑**：渲染循环根据距离上次追踪计算的时间，对 blob 位置进行线性插值，消除 30fps 计算与 60fps 渲染之间的视觉跳变
- **TypedArray 全链路**：所有图像缓冲区使用 `Uint8Array`、`Uint16Array`，避免 GC 压力

#### 3.3 追踪参数
| 参数 | 说明 | 范围 |
|------|------|------|
| 识别阈值 | 差分灵敏度，越小越灵敏 | 20 - 200 |
| 最小面积过滤 | 过滤过小的噪声区域 | 50 - 2000 |
| 跟踪点数量 | 同时追踪的最大 Blob 数 | 1 - 50 |
| 单目标追踪 | 仅追踪面积最大的主体 | 开关 |
| 跳帧设置 | 差分采样步长 | 0 - 5 |

### 📤 5. 导出系统

将带特效的视频导出为本地文件，支持 MP4 与 WebM 格式。

#### 5.1 导出参数
| 参数 | 选项 |
|------|------|
| 格式 | MP4 / WebM（浏览器不支持时自动降级） |
| 分辨率 | 原视频 / 1080P / 720P / 480P |
| 帧率 | 原视频 / 60 / 30 / 24 FPS |

#### 5.2 两阶段精准导出
为解决 `captureStream` 墙钟失真导致的时长错误（如 9 秒视频导出为 48 秒），采用完全解耦的两阶段方案：

**⚙️ 阶段 1：离线逐帧捕获**
- 逐帧 `seek` 到目标时间（`currentTime = i / fps`）
- 精准等待 `seeked` 事件（最多 3 次重试 + 600ms 超时兜底）
- 调用 `updateBlobsForExport()` 计算该帧的追踪状态
- 调用 `drawFrameToTarget()` 渲染完整特效画面
- 缩放至目标分辨率后，使用 `canvas.toBlob()` 压缩为 JPEG Blob（质量 0.94）
- 所有 Blob 存入内存数组，与墙钟完全解耦

**⏱️ 阶段 2：绝对时钟定速推帧**
- 创建 `MediaRecorder`（码率按分辨率×帧率×0.15 计算，最低 16Mbps）
- 从 JPEG Blob 恢复 `ImageBitmap`（GPU 加速解码）
- `drawImage` 到导出 Canvas，调用 `videoTrack.requestFrame()` 推帧
- 使用 `performance.now()` 绝对时钟：下一帧目标时刻 = 阶段开始 + (i+1) × frameIntervalMs
- 若当前已超时则立刻处理下一帧，不累积延迟
- 确保 MediaRecorder 的 PTS 严格等于目标帧间隔

**结果**：输出视频时长 = totalFrames / exportFps ≈ 原视频时长，精确无误。

#### 5.3 导出校验
- 导出前校验：非导出中、已暂停追踪、非摄像头模式、视频时长有效
- 参数超限校验：若导出分辨率/帧率超出原视频规格，弹出确认框提示自动降级
- 进度提示：Toast 通知阶段 1/阶段 2 进度百分比

## 🔄 核心交互逻辑

### 运行时双循环架构

```
用户点击【开始运行】
  ├─→ 启动 _trackLoop（setTimeout，固定 30fps）
  └─→ 启动 mainLoop（rVFC/RAF，匹配视频原生帧率）

_trackLoop（追踪计算循环）:
  ├─→ updateBlobs()
  │   ├─→ 高斯模糊当前帧灰度图
  │   ├─→ 与前一帧差分 → 运动掩码
  │   ├─→ 膨胀操作
  │   ├─→ BFS 轮廓查找 → 原始 Blob 列表
  │   ├─→ NMS 抑制重叠
  │   └─→ 时域平滑（位置/属性插值）
  ├─→ 更新 prevTrackBlobs / currentTrackBlobs
  ├─→ _updateLinkDecisions()（预生成连线/闪线决策）
  └─→ 自适应调度：delay = max(0, 33ms - 本次耗时)

mainLoop（渲染循环）:
  ├─→ 渲染视频帧到 Canvas
  ├─→ 计算插值比例 t = elapsed / 33ms
  ├─→ _getInterpolatedBlobs(prev, curr, t) 位置平滑
  ├─→ _drawLiveOverlay() 绘制所有特效
  │   ├─→ 框内滤镜（从 sourceCanvas 读取像素）
  │   ├─→ 闪线（读取预生成决策）
  │   ├─→ 边框样式
  │   ├─→ 文字标签
  │   ├─→ 中心点
  │   └─→ 连线（读取预生成决策）
  ├─→ 更新进度条
  └─→ 调度下一帧（rVFC 优先，RAF 60fps 降级）
```

### 视频加载流程

```
用户选择视频文件
  └─→ loadVideo(file)
      ├─→ 阶段 1：等待 loadedmetadata（元数据就绪）
      ├─→ 阶段 2：等待 canplaythrough（可流畅播放）
      ├─→ 阶段 3：读取视频参数（宽/高/时长）
      ├─→ 阶段 4：三级帧率检测
      │   ├─→ MP4 Box 解析（前 8MB + 后 12MB）
      │   ├─→ rVFC 中位数法（1x 速，90 帧采样）
      │   └─→ 实时钟帧计数（2.5 秒窗口）
      ├─→ 阶段 5：白名单校验
      │   ├─→ 通过 → 继续加载
      │   └─→ 超规格 → 弹窗降级（4K30 / 1080P60 / 取消）
      └─→ 阶段 6：初始化画布与背景帧缓冲区
```

### 导出流程

```
用户点击【导出 MP4/WebM】
  └─→ startExport(format, resolution, fps)
      ├─→ 前置校验（非导出中 / 已暂停 / 非摄像头 / 时长有效）
      ├─→ 确定导出参数（超出原视频则弹窗确认降级）
      ├─→ ═══════════════════════════════════════
      ├─→ ⚙️ 阶段 1：离线逐帧捕获（0% → 50% 进度）
      │   ├─→ for i = 0 to totalFrames-1
      │   │   ├─→ seek 到 (i-1)/fps 采集 prevFrameGray
      │   │   ├─→ seek 到 i/fps（精准等待 seeked）
      │   │   ├─→ updateBlobsForExport() + _updateLinkDecisions()
      │   │   ├─→ drawFrameToTarget() 渲染特效
      │   │   ├─→ 缩放至目标分辨率
      │   │   └─→ toBlob(JPEG, 0.94) 存入数组
      │   └─→ 每 10 帧让出主线程保持 UI 响应
      ├─→ ═══════════════════════════════════════
      ├─→ ⏱️ 阶段 2：定速推帧编码（50% → 100% 进度）
      │   ├─→ 创建 captureStream + MediaRecorder
      │   ├─→ 预热帧（黑色画面激活流）
      │   ├─→ recorder.start(1)
      │   ├─→ for i = 0 to frameBlobs.length-1
      │   │   ├─→ createImageBitmap(frameBlobs[i])
      │   │   ├─→ drawImage → exportCanvas
      │   │   ├─→ videoTrack.requestFrame()
      │   │   └─→ 绝对时钟等待至 phaseStart + (i+1)*frameIntervalMs
      │   └─→ 等待尾帧刷入 → recorder.stop()
      └─→ 💾 自动下载：BlobTrack_<timestamp>.mp4/webm
```

## 🏆 技术亮点

1. **🔌 零依赖架构**：纯原生 HTML/CSS/JavaScript，无需 npm、构建工具或任何第三方库
2. **⚡ 高性能图像处理**：全链路使用 `TypedArray`（`Uint8Array`、`Uint16Array`、`Float32Array`）处理像素数据，避免 JavaScript 数组的 GC 开销
3. **🔀 分离式渲染管线**：追踪计算（30fps）与视频渲染（原生帧率）完全解耦，通过线性插值保证视觉流畅，计算耗时不会掉帧
4. **🎯 精准帧率检测**：MP4 二进制 Box 解析 + rVFC 中位数法 + 实时钟采样三级降级，精准识别视频真实帧率（包括 NTSC 分数帧率）
5. **📐 精准视频导出**：两阶段解耦方案（离线捕获 + 绝对时钟定速推帧）彻底解决 `captureStream` 墙钟失真问题，输出时长与源视频严格一致
6. **🪟 Liquid Glass UI**：多层 `backdrop-filter` 玻璃态效果，配合动态主题色联动，实现通透、现代、沉浸式的视觉体验
7. **🎲 确定性随机**：Blob 的随机样式/颜色/滤镜通过位置哈希生成，确保同一物体在帧间属性稳定，避免闪烁
8. **🛡️ 超规格防护**：自动检测并提示降级超出设备性能上限的视频，防止卡顿、追踪失效、导出失败
9. **📁 标准项目结构**：HTML / CSS / JS 分离，遵循前端工程最佳实践，便于维护和版本管理

## 🚀 快速开始

### 本地预览

```bash
# 使用 Python 启动本地服务器
python -m http.server 8080

# 或使用 Node.js
npx serve .
```

然后在浏览器中打开 `http://localhost:8080`。

### 直接打开

也可以直接在浏览器中打开 `index.html` 文件（确保 `css/style.css` 和 `js/script.js` 在同一目录下）。部分浏览器的摄像头功能可能需要 HTTPS 或 localhost 环境。

### 部署到静态托管

本项目为纯静态网站，可部署到任何静态托管服务：

- **Netlify**：导入项目文件夹自动部署
- **Vercel**：导入项目文件夹自动部署
- **GitHub Pages**：推送至仓库后开启 Pages 功能
- **Cloudflare Pages / 腾讯云 COS / 阿里云 OSS** 等
